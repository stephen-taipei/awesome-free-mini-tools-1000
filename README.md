## 專案簡介

本專案目標是開發 1000 個純前端小工具，提供實用的線上工具服務。

- **純前端**：無後端伺服器、無資料庫依賴
- **完全離線**：支援 PWA 離線使用
- **隱私保護**：所有資料處理皆在本地完成，零資料外洩風險
- **多國語言**：支援繁體中文、簡體中文、英文、日文、韓文等 10+ 語言
- **高效能**：運用 WebAssembly、WebGPU、Canvas、Web Workers 等先進技術
- **開源免費**：MIT License 完全開放使用修改

## 技術棧

### 基礎框架
- **React 18+**：使用 React 建構 UI 介面
- **TypeScript**：全專案使用 TypeScript 確保型別安全
- **Vite**：使用 Vite 建構與開發工具
- **TailwindCSS**：使用 Tailwind 樣式框架

### 進階技術
- **WebAssembly (WASM)**：用於高效能的圖片處理、影音轉檔等
- **WebGPU**：用於 GPU 加速的 3D 渲染、AI 推論、影像處理等
- **Canvas API**：用於繪圖、圖片編輯
- **Web Workers**：用於背景多執行緒運算，避免阻塞 UI
- **WebRTC**：用於 P2P 連線功能
- **Web Audio API**：用於音訊處理
- **IndexedDB**：用於本地資料儲存
- **Service Worker**：用於離線快取與 PWA

### WASM 模組
- **FFmpeg.wasm**：影音處理
- **Sharp/libvips (WASM)**：圖片處理
- **sql.js**：SQLite in browser
- **pdfjs**：PDF 處理
- **TensorFlow.js / ONNX.js**：機器學習推論

### 狀態管理
- **Zustand**：狀態管理
- **React Query**：資料快取

### 國際化
- **i18next**：多國語言
- **react-i18next**：React 整合

## 使用方式

### 線上使用
直接訪問：`https://[your-domain].com`

### 本地開發

```bash
# 克隆專案
git clone https://github.com/[username]/awesome-free-mini-tools-1000.git

# 進入目錄
cd awesome-free-mini-tools-1000

# 安裝依賴
npm install

# 開發模式執行
npm run dev

# 建構生產版本
npm run build
```

### 支援語言
| 語言代碼 | 語言名稱 | 狀態 |

| zh-TW | 繁體中文 | 主要語言 |
| zh-CN | 简体中文 | 100% |
| en | English | 100% |
| ja | 日本語 | 100% |
| ko | 한국어 | 100% |
| vi | Tiếng Việt | 規劃中 |
| th | ไทย | 規劃中 |
| es | Español | 規劃中 |
| fr | Français | 規劃中 |
| de | Deutsch | 規劃中 |

### 語言檔案格式

```json
{
  "tool": {
    "name": "工具名稱",
    "description": "工具說明",
    "labels": {
      "input": "輸入",
      "output": "輸出",
      "submit": "執行"
    }
  }
}
```

### 新增語言步驟
1. 在 `src/i18n/locales/` 新增對應語言資料夾
2. 複製 `en/` 資料夾作為範本
3. 翻譯所有 JSON 檔案
4. 在 `src/i18n/index.ts` 註冊新語言


### Prompt 模板

#### 首次開發 (從 #1 開始)

```markdown

## 當前進度
- 已完成：#0 (專案初始化)
- 當前任務：#1

## 本次任務
請開發第 #1 個工具：[工具名稱]

### 需求規格
- 工具分類：[文字處理/圖片處理/加密解密 等]
- 功能描述：[具體功能說明]
- 輸入類型：[文字/檔案/數值]
- 輸出類型：[文字/檔案/數值]

### 技術要求
- 純前端實現，無後端依賴
- 支援 PWA 離線使用
- 所有處理在本地完成，保護隱私
- 響應式設計，支援行動裝置

### 完成標準
1. 核心功能實作完成
2. UI 介面完成 (輸入/輸出/操作按鈕)
3. 錯誤處理與使用者提示
4. 本地測試通過
5. 提交 commit 並建立 PR

### PR 格式要求
- 標題：`[#1] 工具名稱 - 工具分類`
- 內容包含：功能說明、使用方式、技術實現、截圖

### 完成後請提供
1. 本次變更摘要 (3-5 點重點)
2. 使用的技術/函式庫說明
3. 下一個工具 #2 的前置準備建議
4. **上下文摘要** (供下次對話使用，約 200-300 字)
```
